# Network Analysis Implementation (Prompt 3)

We have refactored `analyze_network_properties.R` (formerly `cruise_step2a_analyse_network_properties.R`) to include the network analysis and plotting functions.

## Implementation Details

1. **Degree & Strength Distribution Plotting (`plot_degree_dist` & `plot_strength_dist`)**:
   - Uses `igraph` to build undirected graphs and compute degree and strength metrics.
   - Saves figures as PNGs under `data/project_39/figures/` using a **300 DPI** resolution.
   - Saves images specifying dimensions in **inches** (`width = 8`, `height = 8` or `6`) to ensure the fonts and plot elements scale correctly and do not overlap or squish.
   - Distinguishes plots by network type using filename prefixes (`static_` vs. `temporal_`) to prevent overwriting.

2. **Static Network Analysis (`analyze_static`)**:
   - Counts unique nodes using the union of `nodeID_from` and `nodeID_to`.
   - Computes the number of edges (rows) if `static = TRUE`.
   - Appends text summaries to `data/project_39/network_info.txt`.
   - Calls the plotting functions to save `static_degree_distribution.png` and `static_strength_distribution.png`.

3. **Temporal Network Analysis (`analyze_temporal`)**:
   - Counts unique nodes using the union of `nodeID_from` and `nodeID_to`.
   - Computes daily contacts and daily sum of node durations (`2 * sum(contact_duration)`).
   - Generates a dual y-axis chart (`temporal_activity.png`) using `sec_axis` to show contacts (primary axis) and duration (secondary axis) on the same plot.
   - Slices the temporal network into an aggregated static representation to call the plotting functions (saves `temporal_degree_distribution.png` and `temporal_strength_distribution.png`).
   - Appends text summaries to `data/project_39/network_info.txt`.

4. **Auto-Execution Block**:
   - An execution block is added at the end of the script to automatically run both analyses on `static_network_sail_1.csv` and `temporal_network_sail_1.csv` when executed via `Rscript` (non-interactively).
