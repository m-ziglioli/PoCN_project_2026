# Analysis of R Scripts in project_39

This document provides a comprehensive overview of the purpose, inputs, internal logic, and output files generated by each R script in the `project_39` codebase.

---

## 1. Network Preparation & Configuration

### `cruise_load_library.R`
- **Purpose**: Loads all required R libraries for data manipulation, graph/network processing, parallel computing, and plotting.
- **Key Libraries**: 
  - Data: `dplyr`, `purrr`, `readr`, `data.table`
  - Plotting/Graphing: `igraph`, `grid`, `ggplot2`, `diagram`
  - Calculations: `sn` (skew-normal distribution), `clue` (linear assignment solver), `rdist` (distances), `doParallel` (parallel processing)
- **Output**: None directly (loads libraries into the R environment).

### `cruise_plot_colour.R`
- **Purpose**: Defines a unified design and color palette (using the Nature theme) for network and simulation plotting.
- **Logic**: Defines custom translucent colors (`CONFIG$colsLight1`, `CONFIG$colsLight2`, `CONFIG$colsLight3`) based on alpha values.
- **Output**: None directly (defines plot styling variables in memory).

## 2. Network Reconstruction and Metrics

### `cruise_step1_generate_edgelist_weighted.R`
- **Purpose**: Reconstructs the cruise contact networks and calibrates transmission coefficients (FOI) for the epidemic model under different interventions.
- **Inputs**: 
  - `input_data/dataNodes.RData`
  - `input_data/dataContact.RData`
- **Internal Logic**:
  1. Filters nodes and edges for a specific sailing (default `sail = 1`).
  2. Classifies contact pairs into 6 categories (0: Passengers in same cabin; 1: Passengers in same traveling group but different cabins; 2: Passengers in different traveling groups; 3: Close crew colleagues; 4: Non-close crew colleagues; 5: Passenger-to-crew contacts).
  3. Formulates a **temporal network** (daily contact durations) and a **static network** (mean contact duration across the cruise duration).
  4. Generates three weighting schemes:
     - `edgeList.trans.long`: 95% saturation after 3 hours of cumulative contact.
     - `edgeList.trans.short`: 95% saturation after 1 hour of cumulative contact.
     - `edgeList.prob`: Probabilistic weight representation.
  5. Calibrates transmission scaling factors using `uniroot` to target a baseline household attack rate (AR) of 20% under various intervention scenarios (no interventions, masks, vaccination combos).
- **Outputs**:
  - `data/edgelist weighted/edgeListTemporal_sail_1.Rdata`
  - `data/edgelist weighted/edgeListStatic_sail_1.Rdata`
  - `data/foi/foi_scale_set_1_sail_1.Rdata`

### `cruise_step2a_analyse_network_properties.R`
- **Purpose**: Analyzes the structural network characteristics of the reconstructed static cruise contact networks.
- **Inputs**:
  - `input_data/dataNodes.RData`
  - Static weighted networks generated in Step 1.
- **Internal Logic**:
  1. Loops over sailings 1 to 4 to load static networks.
  2. Constructs undirected `igraph` network representations.
  3. Computes node-level properties: degree, strength (weighted degree), eigenvector centrality, betweenness centrality, and local clustering coefficient (`CC`).
  4. Identifies the connected components and labels nodes with their corresponding component ID.
- **Output**:
  - `data/dataNodes_test.RData` (updated node data with structural metrics).

### `cruise_step2b_analyse_demographics.R`
- **Purpose**: Analyzes the demographic structure of the cruise cohorts and tests differences in network properties between passengers and crew.
- **Inputs**:
  - `data/dataNodes.RData` (includes the calculated network properties).
- **Internal Logic**:
  1. Summarizes demographics (ages, age groups, genders for passengers; departments/cohorts for crew).
  2. Extracts medians and IQRs of network properties (strength, eigenvector centrality, local clustering coefficient) for passengers vs. crew.
  3. Performs Welch's two-sample t-tests to evaluate the statistical significance of difference in network position between passengers and crew.
- **Output**: Prints summary statistics and t-test summaries to the console.

### `cruise_step2c_plot_network_properties.R`
- **Purpose**: Visualizes node-level network properties (degree, eigenvector centrality, and clustering coefficient) grouped by department cohorts and sailings.
- **Inputs**:
  - `data/dataNodes.RData`
- **Logic**: Generates a 3-panel figure showing median, 50% CI (IQR), and 95% CI values for each cohort (e.g. F&B, Galley, Marine, Passengers) across all four sailings.
- **Output**:
  - `figure/graph.png` (high-resolution composite plot image).

### `cruise_step2d_plot_network.R`
- **Purpose**: Generates high-fidelity circular grid visualizations of the contact networks for each department cohort.
- **Inputs**:
  - `data/dataNodes.RData`
  - `data/edgelist weighted/edgeListStatic_sail_1.RData`
- **Logic**: Defines `draw.network` which leverages circular coordinates mapping (using `solve_LSAP` to minimize crossing lines) and colors nodes based on their cohort category.
- **Outputs**:
  - `figure/nature comms pdf/s2_networks_seed321_all_pass.pdf`
  - `figure/networks/networks_seed321.svg`

## 3. Epidemic Spreading Simulations

### `outbreak_step3a_generate_onset.R`
- **Purpose**: Models the conditional symptom onset distribution for index cases.
- **Logic**: Combines a log-normal incubation period (mean 4 days) with a uniform probability of infection before boarding. Filters out cases showing symptoms prior to boarding, yielding the conditional probability distribution of symptom onset on days 1–3 (`distOnset.3` for temporal network) and days 1–7 (`distOnset.7` for static network).
- **Outputs**:
  - `data/onset/distOnset.3.RData`
  - `data/onset/distOnset.7.RData`

### `outbreak_step3b_param.R`
- **Purpose**: Generates parameter sets to simulate epidemic processes across experimental scenarios.
- **Logic**: Defines combinations of interventions (nothing vs. isolation), testing regimes (none, PCR once, Antigen twice), vaccination coverages (0% to 100%), and masking. Creates separate parameters for the static (7 days duration) and temporal (3 days duration) network setups.
- **Output**: Defines scenario data tables in memory (`paramStatic`, `paramTemporal`).

### `outbreak_step3c_functions.R`
- **Purpose**: Implements core mathematical functions for epidemic transmission probabilities and test sensitivities.
- **Logic**:
  - `inf_prob`: Computes the daily transmission probability along an edge, incorporating skew-normal serial intervals, presymptomatic transmission rate, force of infection, contact duration, and asymptomatic infectivity reduction.
  - `PCR_prob` & `Ag_prob`: Computes diagnostic test sensitivities on a given day based on a Ct-value/viral load trajectory spline model (`ct.traj`).
- **Output**: Utility functions in memory.

### `outbreak_step3d_setup.R`
- **Purpose**: Initializes the branching process population.
- **Logic**: Seeds initial cases ("I" status), sets vaccination status, and applies pre-departure screening tests to isolate cases before the sailing starts.
- **Output**: Returns the initial population state dataframe.

### `outbreak_step3e_step.R`
- **Purpose**: Steps the branching model forward by one day.
- **Logic**:
  - Updates isolation, quarantine, and recovery states.
  - Adds external infections (if outside force of infection is non-zero).
  - Identifies contacts of infectious individuals.
  - Evaluates transmission along active edges using joint vaccine/mask efficacy scale factors.
  - Generates new infections and manages contact tracing.
- **Output**: Updates the population state dataframe for the next step.

### `outbreak_step3f_model.R`
- **Purpose**: Runs a single stochastic simulation run of the epidemic model.
- **Logic**: Loops `outbreak_step` until the epidemic goes extinct, infects the entire population, or reaches the maximum simulation days.
- **Output**: Returns daily, weekly, or raw case data from a simulation run.

### `outbreak_step3g_sim_parallel.R`
- **Purpose**: Wrapper to run multiple stochastic realizations of a scenario parameter set.
- **Logic**: Maps human-readable interventions to simulation options, supports network structural modifications (e.g. null networks, distancing), and runs the model parallelly.
- **Output**: Returns a combined dataframe of all cases across the parallelized runs.

### `outbreak_step3h_simulate.R`
- **Purpose**: Orchestrates the multi-scenario parallelized simulation.
- **Logic**: Merges calibrated FOI values, configures the parallel worker cluster (`doParallel`), runs simulations over all parameter/scenario combinations, and collects results.
- **Output**:
  - `data/outputs/resultsTemporal_set_19.RData`

### `outbreak_step3i_plot_outbreak_network.R`
- **Purpose**: Visualizes transmission pathways of a specific simulated outbreak.
- **Logic**: Extracts a single outbreak run (e.g., paramset 27, baseline scenario), highlights infected nodes, and draws curved arrows tracing transmissions from infector to infectee.
- **Outputs**:
  - `figure/nature comms pdf/m4a_outbreak_arr.pos_0.96_seed123.pdf`
  - `figure/networks/outbreak_arr.pos_0.96_seed123.svg`

### `outbreak_step3j_plot_simulate.R`
- **Purpose**: Summarizes and plots aggregate results from all simulation outputs.
- **Logic**: Loads all RData outputs, aggregates indicators (outbreak size distribution, cumulative incidence, generation length), and creates comparative graphs.
- **Outputs**: High-fidelity SVG/PDF figures in `figure/`.
