Here is the explanation of the mathematical formulation and how to model your dataset:

### 1. Mathematical Formulation of Agent-Based SEIR Model

In an Agent-Based Model (ABM), the population consists of discrete agents $i \in \{1, \dots, N\}$. Each agent at any time step $t$ is in one of four states:
$$\mathcal{S}_i(t) \in \{S, E, I, R\}$$

Unlike compartmental ODE models where transition rates apply to the whole population, transitions in ABM are probabilistic at the agent level:

1. **Susceptible to Exposed ($S \to E$):**
   A susceptible agent $i$ ($S$) transitions to exposed ($E$) based on its interactions with infectious neighbors $j$ ($I$) during day $t$.
   The probability that agent $i$ is infected by agent $j$ during an interaction of duration $w_{ij}(t)$ is:
   $$p_{ij}(t) = 1 - e^{-\beta \cdot w_{ij}(t)}$$
   where $\beta$ is the transmission rate per unit of interaction time.
   
   The probability that agent $i$ escapes infection from all infected partners $N_i^{\text{inf}}(t)$ on day $t$ is:
   $$P(\text{escape}_i(t)) = \prod_{j \in N_i^{\text{inf}}(t)} (1 - p_{ij}(t)) = e^{-\beta \sum_{j \in N_i^{\text{inf}}(t)} w_{ij}(t)}$$
   
   Thus, the probability of transition $S \to E$ for agent $i$ on day $t$ is:
   $$P(S_i \to E_i) = 1 - e^{-\beta \sum_{j \in N_i^{\text{inf}}(t)} w_{ij}(t)}$$

2. **Exposed to Infectious ($E \to I$):**
   An agent remains in the Exposed state ($E$) for a latent period $d_E$. This can be modeled as:
   - A fixed duration: $d_E$ days.
   - A probabilistic daily transition: $P(E \to I) = \sigma = 1 / D_E$, where $D_E$ is the average latent period.

3. **Infectious to Recovered ($I \to R$):**
   An infectious agent ($I$) recovers after an infectious period $d_I$:
   - A fixed duration: $d_I$ days.
   - A probabilistic daily transition: $P(I \to R) = \gamma = 1 / D_I$, where $D_I$ is the average infectious period.

### 2. How to Model Multiple Daily Interactions in Your Dataset

Your dataset represents a dynamic, weighted contact network. Since there are multiple interactions between the same pair of nodes (and different pairs) in a day, you can aggregate them or model them step-by-step:

#### Option A: Daily Aggregation (Recommended for Daily Time-Steps)
If the SEIR simulation runs in daily time steps, you can aggregate the interaction durations for each day $t$:
1. Group your data by `day`, `node_from`, and `node_to`.
2. Sum the `interaction duration` for each pair of nodes to get the total daily contact weight $w_{ij}(t)$:
   $$w_{ij}(t) = \sum_{\text{interactions on day } t} \text{duration}(i, j)$$
3. If the network is undirected, ensure $w_{ij}(t) = w_{ji}(t)$ by summing interactions in both directions.
4. Calculate the daily transmission probability for agent $i$ using the aggregated weights:
   $$P(S_i \to E_i) = 1 - \exp\left(-\beta \sum_{j \in N_i^{\text{inf}}(t)} w_{ij}(t)\right)$$

#### Option B: Event-Driven / Fine-Grained Simulation
If you need to model the precise sequence of events (e.g. infection happening mid-day):
1. Order interactions chronologically within each day.
2. For each interaction between $i$ and $j$ of duration $\Delta t$:
   - If one is $S$ and the other is $I$, transition the $S$ node to $E$ with probability $1 - e^{-\beta \Delta t}$.
